## 카테고리별 수집 내용

### [카테고리 1] 배경 및 목적

**Q1 (왜 만드는가/한계)**: 020-data-migration-cutover 는 `files.files` 테이블의 **메타데이터**(id·key·url·contentType·size 등)만 이관 대상에 포함하고(FR-017/SC-019), 실 파일 바이너리(오브젝트) 이관은 "R2 실연동 완료 후" 를 전제로 범위 외로 명시 이월했다(020 spec.md "범위 외" 절, 020 CHANGES.md 후속 주의사항 #3). 021-payment-file-integration 이 `R2FileStorage`(Cloudflare R2 실연동, `@aws-sdk/client-s3` presigned URL)를 완성하여 이 전제가 충족되었다(021 CHANGES.md 후속 주의사항 #6). 현재 레거시 AWS(S3 등)에 있는 실제 파일 바이너리(상품 이미지·리뷰 이미지·프로필 등)는 신규 R2 버킷에 존재하지 않으므로, `files.files` 메타데이터만 이관된 상태로 컷오버하면 `url` 필드가 가리키는 실제 객체가 신규 인프라에 없어 파일 접근이 깨진다.

**Q2 (기존 해결 방식·한계)**: 현재는 해결 방식 자체가 부재 — 020 이 명시적으로 범위 외로 이월했으며 별도 spec 미생성 상태.

**Q3 (성공 판단 기준)**: 사용자가 Q7-2(검증 기준: 개수 100% + 샘플 체크섬)를 확정함에 따라 자연스럽게 확정됨 — "컷오버 완료 후 `files.files`(status=UPLOADED) 레코드가 가리키는 실 파일 바이너리 전건이 R2 에 존재하고(개수 100% 일치), 무작위 샘플의 체크섬이 레거시와 일치"를 성공 기준으로 확정(FR-005/SC-005 로 구체화).

### [카테고리 2] 사용자 & 이해관계자

**Q4 (누가 사용하는가)**: 파일 소유자(구매자·판매자 — 상품 이미지·리뷰 이미지·프로필 사진 업로더), 플랫폼 운영자(이관 실행 주체), 최종 구매자(상품/리뷰 이미지 열람자).

**Q5 (기술 수준)**: 최종 사용자(구매자·판매자)는 이관 과정을 인지하지 않음(투명 이관). 운영자는 020 과 동일하게 CLI·스크립트 실행 가능 수준으로 가정.

**Q6 (이해관계자)**: 020/021 과 동일 — 플랫폼 운영자, 개발팀. 신규 이해관계자 없음.

### [카테고리 3] 핵심 기능

- Q3-1: `status=UPLOADED` 레코드만 이관 대상(A 채택). PENDING 은 대상 제외.
- Q3-2: 사전 대량복사(시간 무관) + 컷오버 윈도우 내 최종 델타만 처리(B 채택).
- Q3-3: 서버사이드 스크립트(AWS SDK 또는 `aws s3 sync`/`rclone` 표준 CLI, A 채택).
- Q3-4: 레거시 key 를 R2 객체 key 로 그대로 사용, `url` 필드만 R2 공개 URL 로 갱신(A 채택). 리키잉 없음.

### [카테고리 4] 데이터 & 입출력

- Q4-1: 레거시 S3 접근은 파이프라인 밖(사용자 환경) — 옵션 A(산출물 절차 제시 → 사용자 실행 → 결과 전달 → 검증, 020 승계) 채택.
- 주요 데이터: 실 파일 바이너리(상품/리뷰/프로필 이미지 등, 011-file-security 4종 MIME allowlist 범위).
- 외부 연동: 레거시 AWS S3(source, read-only) → Cloudflare R2(target).
- 민감도: 프로필 이미지 등 개인 식별 가능 콘텐츠 포함 가능 — 전송 채널 암호화(TLS/HTTPS) 필요(NFR-004).

### [카테고리 5] 제약조건

- Q5-1: 구체적 규모 추정치 없음 — 사전 실측(precheck) 단계 신설(FR-009).
- Q5-2: 사전 대량복사 단계는 명확한 시간 제한 없음(여유 있게 진행). 컷오버 윈도우 내 델타 구간만 020 NFR-001(60분) 승계.

### [카테고리 7] 예외 & 실패 시나리오

- Q7-1: 개별 파일 실패는 실패 목록 기록 + 재시도(부분 성공 허용, 전체 중단 없음). 단 사전 복사 완료 시점(컷오버 개시 전)까지 잔존하는 실패는 사전 평가 리포트에 포함하여 사용자 재확인 요청(FR-007, 020 FR-012 패턴 승계).
- Q7-2: 검증은 개수 100% 일치 + 무작위 샘플 체크섬 비교(020 SC-007 패턴 승계). 전수 체크섬은 채택하지 않음(시간 비용 과다).
