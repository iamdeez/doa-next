## 배경 및 목적

009-flutter-customer-app Phase 1 MVP 구현 시 일정·범위 최적화를 위해 명시적으로 Out of Scope으로 이월한 기능들을 Phase 2로 구현한다.

이월 항목(009 spec.md §범위 외 기준):
- CategoryScreen 하드코딩 카테고리 목록 → 서버 API 연동
- 마이페이지 6개 항목(1:1문의·FAQ·공지사항·알림설정·마일리지·개인정보수정) 미연결
- 마이페이지 프로필 하드코딩 → 실 사용자 데이터
- 비밀번호 재설정 / 아이디 찾기 비활성 링크
- 소셜 로그인(카카오·구글·네이버) — 본 spec에서 014로 추가 이월

Phase 2 완료 후 소비자 앱의 핵심 자가 서비스(계정 복구, 정보 수정, 고객 지원 접근) 기능이 갖춰진다.

## 기능 요구사항

### 카테고리 API 연동

- **FR-001**: 카테고리 화면 진입 시 서버로부터 카테고리 목록을 조회하여 동적으로 표시한다. 기존 하드코딩 목록은 제거한다.
- **FR-002**: 카테고리 목록 조회 실패 시 오류 안내 메시지와 재시도 수단을 제공한다.

### 마이페이지 프로필 실데이터 연동

- **FR-003**: 마이페이지 화면 표시 시 현재 로그인된 사용자의 실제 이름과 이메일 주소를 표시한다. 기존 하드코딩 문자열("내 계정", "user@email.com")을 제거한다.

### 개인정보수정

- **FR-004**: 개인정보수정 화면에서 이름과 연락처를 수정하여 저장할 수 있다. 저장 성공 시 수정된 값이 화면에 반영된다.

### 고객 서비스 접근

- **FR-005**: 1:1 문의하기 항목 선택 시 앱에 설정된 고객지원 이메일 주소를 수신처로 하여 기기 이메일 앱을 통해 문의를 작성할 수 있다. 고객지원 이메일 주소는 앱 내 구성 가능한 상수로 관리하며 특정 주소를 하드코딩하지 않는다.
- **FR-006**: FAQ 화면에서 사전 정의된 자주 묻는 질문 항목 목록을 조회한다.
- **FR-007**: 공지사항 화면에서 사전 정의된 공지 항목 목록을 조회한다.

### 알림 설정

- **FR-008**: 알림 설정 화면에서 알림 유형별(주문·배송·이벤트 등) 수신 여부를 설정하고 앱 재시작 후에도 설정이 유지된다.

### 마일리지

- **FR-009**: 마일리지 포인트 화면에서 서비스 준비 중임을 안내하는 화면을 표시한다.

### 비밀번호 재설정

- **FR-010**: 로그인 화면의 "비밀번호 재설정" 경로를 통해 비밀번호 재설정 요청 화면으로 이동할 수 있다.
- **FR-011**: 비밀번호 재설정 요청 화면에서 가입 이메일 주소를 입력하면 해당 이메일로 6자리 숫자 OTP를 발송한다.
- **FR-012**: OTP 입력 화면에서 OTP와 새 비밀번호를 입력하여 비밀번호를 변경한다. 변경 성공 시 로그인 화면으로 이동한다.
- **FR-013**: OTP 재발송 요청은 이전 발송 요청 후 1분 이내에는 허용하지 않는다.

### 이메일 찾기

- **FR-014**: 로그인 화면의 "아이디 찾기" 항목 명칭을 "이메일 찾기"로 변경하고, 해당 경로를 통해 이메일 찾기 화면으로 이동할 수 있다.
- **FR-015**: 이메일 찾기 화면에서 가입 시 등록한 전화번호를 입력하면 해당 전화번호에 연결된 가입 이메일을 마스킹된 형태로 반환한다.
- **FR-016**: 이메일 찾기 결과 이메일은 마스킹된 형태로 표시한다.

## 비기능 요구사항

- **NFR-001**: 카테고리 목록 화면 표시 완료 시간 P95 3초 이내.
- **NFR-002**: 비밀번호 재설정 OTP 유효 시간은 발급 후 10분이며, 만료된 OTP는 인증에 사용할 수 없다.
- **NFR-003**: 동일 이메일 주소에 대한 OTP 발송 요청은 1분당 최대 1회를 초과할 수 없다.
- **NFR-004**: 이메일 찾기 결과의 이메일 주소는 앞 2자를 제외한 @ 앞 부분을 마스킹(**) 처리하여 표시하고, 도메인 부분은 그대로 표시한다.
- **NFR-005**: `flutter analyze` 명령 실행 결과 0 issues (009 기준 유지).

- **SC-002** (`FR-001` 관련): CategoryScreen에 하드코딩된 카테고리 문자열 목록이 존재하지 않는다. `[env:static]`
- **SC-003** (`FR-002` 관련): GET /categories 호출 실패 시 오류 메시지와 재시도 버튼이 화면에 표시된다. `[env:unit]`

### 마이페이지 프로필 실데이터 연동

- **SC-004** (`FR-003` 관련): MyPageScreen 표시 시 GET /auth/me API가 호출되고 응답의 사용자 이름과 이메일이 화면에 표시된다. `[env:unit]`
- **SC-005** (`FR-003` 관련): MyPageScreen에 하드코딩된 프로필 문자열("내 계정", "user@email.com")이 존재하지 않는다. `[env:static]`

### 개인정보수정

- **SC-006** (`FR-004` 관련): 개인정보수정 화면에서 이름·연락처 변경 후 저장 시 PATCH /users/me가 호출되고, 성공 응답 시 변경된 값이 화면에 반영된다. `[env:unit]`
- **SC-007** (`FR-004` 관련): PATCH /users/me 실패 시 오류 메시지를 표시하고 입력값을 유지한다. `[env:unit]`

### 고객 서비스 접근

- **SC-008** (`FR-005` 관련): 1:1 문의하기 항목 선택 시 고객지원 이메일 상수를 수신처로 한 이메일 작성 화면이 실행된다. `[env:unit]`
- **SC-009** (`FR-005` 관련): 고객지원 이메일 주소는 코드 내 구성 가능한 상수로 관리되며 특정 이메일 주소 리터럴이 하드코딩되어 있지 않다. `[env:static]`
- **SC-010** (`FR-006` 관련): FAQ 화면에서 정적 FAQ 항목이 1개 이상 표시된다. `[env:static]`
- **SC-011** (`FR-007` 관련): 공지사항 화면에서 정적 공지 항목이 1개 이상 표시된다. `[env:static]`

### 알림 설정

- **SC-012** (`FR-008` 관련): 알림 설정 화면에서 알림 유형별 on/off 스위치가 표시되고, 설정 변경 후 앱 재진입 시 변경된 설정이 복원된다. `[env:unit]`

### 마일리지

### 비밀번호 재설정

- **SC-018** (`FR-012`, `NFR-002` 관련): 발급 후 10분이 경과한 OTP로 POST /auth/reset-password 요청 시 오류 응답(4xx)을 반환한다. `[env:integration]`

## 범위 외

- **소셜 로그인(카카오·구글·네이버 OAuth)**: 백엔드 OAuth 인프라 규모로 별도 014 spec으로 분리. LoginScreen 소셜 버튼은 현 플레이스홀더 UI 상태 유지(onTap 미구현).
- **알림설정 백엔드 연동**: 본 spec에서는 기기 로컬 저장만 구현. 서버 알림 제어는 추후 spec.
- **FAQ·공지사항 백엔드 API**: 정적 콘텐츠만 제공. 서버 기반 CMS 연동은 추후 spec.
- **마일리지 실 기능**: "서비스 준비 중" 안내 화면만 제공. 마일리지 적립·사용·조회는 추후 spec.
- **로그인 상태에서 비밀번호 변경**: 개인정보수정 화면에서 비밀번호 변경 불포함. PATCH /users/me는 이름·연락처만 다룸.
- **이메일 발송 서비스 선택**: 이메일 OTP 발송을 위한 SMTP/서비스 선택은 HOW 결정이므로 Planning 단계에서 확정.
- **최근 본 상품 마이페이지 연결**: 013 범위 외. MyPageScreen 해당 항목 현 상태(onTap: null) 유지.
- **개인정보수정 시 이메일 변경**: UpdateProfileDto가 name·phone만 지원함. 이메일 변경은 별도 검토 필요.

**사후 운영 검증 피드백 사이클**:

본 spec 파이프라인 종료 후 운영 환경에서 점검할 시나리오:
1. 비밀번호 재설정 플로우 실 이메일 발송 및 OTP 수신 확인
2. OTP 만료(10분 경과) 후 reset 거부 동작 확인
3. 전화번호 기반 이메일 찾기 실 데이터 검증
