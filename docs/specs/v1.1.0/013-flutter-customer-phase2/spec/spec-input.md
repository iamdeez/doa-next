## 카테고리별 수집 내용

### [카테고리 1] 배경 및 목적

**Q1. 이 시스템/기능을 왜 만드는가?**
009-flutter-customer-app Phase 1 MVP 구현 시 명시적으로 Out of Scope으로 이월한 기능들을 Phase 2로 구현한다.
이월 근거: 009 spec.md §범위 외(카테고리 API 연동·마이페이지 미연결·비밀번호 재설정·아이디 찾기).
소셜 로그인은 009→013 이월 후 013→014로 재이월(OAuth 인프라 규모).

**Q2. 현재 어떻게 해결하고 있는가? (기존 방식의 한계)**
- CategoryScreen: 10개 하드코딩 항목 — 서버 카테고리 데이터와 불일치 가능
- MyPageScreen 프로필: "내 계정" / "user@email.com" 하드코딩 — 실 사용자 정보 미반영
- MyPageScreen 항목: 6개 모두 onTap: null — 탭해도 동작 없음
- 소셜 로그인 버튼: UI만 존재, 동작 없음(→ 014 해결)
- 비밀번호 재설정·아이디 찾기: _LinkRow 텍스트만 존재, 비활성

**Q3. 성공 판단 기준**
- CategoryScreen이 GET /categories 응답 실데이터로 렌더링됨
- MyPageScreen 6개 항목(1:1문의·FAQ·공지사항·알림설정·마일리지·개인정보수정) 탭 시 각각 동작 실행
- MyPageScreen 프로필에 실 사용자 이름·이메일 표시
- 비밀번호 재설정 플로우 완주 가능 (이메일 입력→OTP 발송→OTP 입력→비밀번호 변경)
- 이메일 찾기 동작 (전화번호 입력→마스킹 이메일 반환)
